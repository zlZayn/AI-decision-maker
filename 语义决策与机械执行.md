# AI 决策 + 脚本执行：轻量级数据处理框架

---

## 一、核心问题

当前数据处理面临的困境：

| 问题类型 | 表现 | 原因 |
|---------|------|------|
| **规则困境** | 写大量 if-else，难以覆盖所有情况 | 语义理解超出规则能力 |
| **AI 困境** | token 消耗高，输出不可控 | 没有明确的决策边界 |
| **协作困境** | AI 做完后脚本无事可做 | 分工不明确 |

---

## 二、设计原则

1. **AI 做语义理解**：判断"是什么"、"什么意思"
2. **脚本做标准化执行**：执行验证、转换、存储等确定操作
3. **接口清晰**：AI 输出固定格式，脚本按格式执行

---

## 三、精妙场景

### 场景 1：分类变量有序/无序判断

**问题**：脚本无法判断"学历"是有序变量（小学<初中<高中），而"性别"是无序变量。

**AI 输入**：
```
字段名："学历"
样本值：["小学", "初中", "高中", "本科"]
```

**AI 输出**：
```json
{
  "variable_type": "categorical",
  "subtype": "ordinal",
  "order": ["小学", "初中", "高中", "本科"],
  "confidence": 0.95
}
```

**脚本执行**：
```python
if result["subtype"] == "ordinal":
    # 使用非参数检验（如 Kruskal-Wallis）
elif result["subtype"] == "nominal":
    # 使用卡方检验
```

---

### 场景 2：缺失值处理策略选择

**问题**：不同数据类型需要不同的缺失值处理策略。

**AI 输入**：
```
字段名："年龄"
缺失率：25%
样本值：[28, 35, null, 42, null]
```

**AI 输出**：
```json
{
  "strategy": "median_imputation",
  "reason": "连续变量，缺失率中等，使用中位数填充"
}
```

**脚本执行**：
```python
if result["strategy"] == "median_imputation":
    df["年龄"].fillna(df["年龄"].median(), inplace=True)
```

---

### 场景 3：非结构化文本映射

**问题**：将简历文本映射到数据库字段。

**AI 输入**：
```
文本："张三，男，28岁，电话138xxxx"
可用字段：user_name, gender_code, age_num, phone_n
```

**AI 输出**：
```json
{
  "user_name": "张三",
  "gender_code": "M",
  "age_num": 28,
  "phone_n": "138xxxx"
}
```

**脚本执行**：
```python
# 验证字段格式
validate_phone(result["phone_n"])
# 存储到数据库
db.insert(result)
```

## 四、核心设计

### 4.1 字段注册表

脚本维护的字段定义：
```python
REGISTRY = {
  "phone_n": {"type": "phone", "desc": "手机号"},
  "gender_code": {"type": "enum", "values": ["M", "F"]},
  "age_num": {"type": "int", "desc": "年龄"}
}
```

### 4.2 工作流

```
输入 → AI 判断 → 输出标签/映射 → 脚本执行
   ↓
字段注册表（脚本维护）
```

### 4.3 接口规范

| 场景 | AI 输出格式 | 脚本处理 |
|------|-----------|---------|
| 分类变量判断 | `{"subtype": "ordinal"}` | 选择统计方法 |
| 缺失值处理 | `{"strategy": "median_imputation"}` | 执行填充 |
| 文本映射 | `{"field_name": "value"}` | 验证存储 |

---

## 五、优势总结

| 维度 | AI 做什么 | 脚本做什么 |
|------|----------|-----------|
| 语义理解 | 判断有序/无序 | - |
| 策略选择 | 选择处理策略 | 执行策略 |
| 格式映射 | 文本→字段 | 验证存储 |
| Token 消耗 | 低（只看样本） | - |
